# Kanji Practice

Веб-приложение для изучения японских иероглифов с анализом написания черт в реальном времени и системой интервального повторения.

## Обзор

Kanji Practice помогает изучающим японский язык освоить правильное написание иероглифов через интерактивную практику. Приложение сравнивает нарисованные пользователем черты с эталонными данными из KanjiVG, используя алгоритм Dynamic Time Warping (DTW), и выдаёт мгновенную обратную связь по каждой черте.

## Текущие возможности

### Практика написания
- Интерактивный холст для рисования отдельных черт иероглифа
- Сравнение с эталонными траекториями из базы KanjiVG в реальном времени
- Оценка каждой черты по форме, углу наклона и длине
- Цветовая индикация правильно и неправильно нарисованных черт
- Фильтрация иероглифов по уровням JLPT (N5-N1)
- Последовательный режим с автоматическим переходом к следующей черте

### Словарь иероглифов
- Поиск по 2136 иероглифам с чтениями и значениями
- Отображение онъёми (китайские чтения) и кунъёми (японские чтения)
- Английские значения и уровни JLPT
- Информация о количестве черт
- Переход к практике конкретного иероглифа из словаря

### Система заданий
- Три типа вопросов: угадывание чтения, угадывание значения, выбор иероглифа
- Фильтрация вопросов по уровню JLPT
- Обратная связь с отображением правильного ответа
- Интеграция со словарём для показа детальной информации после ответа

### Уроки
- Структурированные уроки с последовательным введением новых иероглифов
- Встроенный холст для практики письма внутри урока
- Мини-викторины в составе уроков
- Отслеживание прогресса с сохранением в localStorage
- Возможность перепрохождения завершённых уроков
- Индикатор прогресса урока

### Хранение данных
- База данных SQLite для хранения информации об иероглифах и прогрессе пользователя
- Запись истории практики (оценки черт, расстояния DTW)
- Кеширование соответствий иероглифов и SVG-файлов для быстрого доступа
- Определения уроков в формате JSON

## Архитектура

### Сервер (Go)
Сервер построен на стандартной библиотеке Go `net/http` и обеспечивает:
- Раздачу статических файлов (HTML, CSS, JavaScript)
- REST API для практики, словаря, заданий и уроков
- Слой доступа к базе данных SQLite

### База данных (SQLite)
Схема включает таблицы:
- `kanji_characters` — эталонные данные иероглифов (чтения, значения, уровень JLPT, количество черт)
- `users` — учётные записи пользователей с хешированными паролями
- `user_progress` — прогресс изучения по каждому иероглифу (статус, уровень SRS, дата повторения, количество ошибок)
- `practice_history` — детальная история практики (оценка черты, DTW-расстояние, разница углов и длин)
- `lessons` — структура уроков
- `vocabulary` — примеры слов с чтениями и переводами

### Ядро анализа черт (пакет stroke)
- `stroke.go` — базовые типы: Point, Stroke, KanjiTemplate, StrokeResult, AnalysisResult
- `comparator.go` — алгоритм DTW, сравнение углов и длин, вычисление общей оценки
- `kvg_loader.go` — парсинг SVG-файлов KanjiVG с интерполяцией кривых Безье, нормализация направления черт

### Фронтенд
- Навигация между разделами (главная, практика, словарь, задания, уроки)
- Интерактивный canvas с поддержкой мыши и тач-устройств
- Масштабирование координат из пространства SVG (109x109) в пространство canvas (400x400)
- Сглаживание пользовательского ввода
- Кеширование списка иероглифов на клиенте

## API

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/kanji` | Список иероглифов (опциональный фильтр `?level=N5`) |
| GET | `/api/template/{id}` | SVG-файл эталона по идентификатору |
| POST | `/api/check-stroke` | Проверка одной нарисованной черты |
| POST | `/api/check` | Проверка всех черт иероглифа |
| GET | `/api/dictionary/{char}` | Информация об иероглифе (чтения, значение) |
| GET | `/api/quiz` | Вопрос для викторины (параметры: `?type=reading&level=N5`) |
| GET | `/api/lessons` | Список уроков |
| GET | `/api/progress/` | Прогресс пользователя по всем иероглифам |
| GET | `/api/progress/{kanji_id}` | Прогресс по конкретному иероглифу |
| GET | `/api/reviews/due` | Иероглифы для повторения (защищённый) |
| POST | `/api/reviews/submit` | Отправка оценки повторения (защищённый) |

## Алгоритм сравнения черт

1. Парсинг SVG-файла KanjiVG с извлечением траекторий через интерполяцию кривых Безье (20 шагов на сегмент)
2. Нормализация направления эталонных черт (всегда сверху вниз, слева направо)
3. Центрирование обеих траекторий (вычитание центра масс)
4. Нормализация размера до единичного ограничивающего прямоугольника
5. Вычисление DTW-расстояния между эталонной и пользовательской траекториями
6. Сравнение средневзвешенного угла и общей длины
7. Итоговая оценка: форма (50%), длина (30%), угол (20%)
8. Порог принятия черты: 70% (в уроках) или 80% (в свободной практике)

## Запуск

```bash
# Установка зависимостей
go mod tidy

# Запуск сервера
go run main.go
```

При первом запуске приложение:
1. Создаёт файл базы данных `data/kanji.db`
2. Выполняет SQL-схему из `db/schema.sql`
3. Парсит SVG-файлы из `model/kanji/` и сохраняет соответствия в БД
4. Загружает начальные данные словаря из `model/kanji_dict.json`

При последующих запусках сидирование пропускается, кеш SVG загружается из БД.

Сервер запускается на `http://localhost:8080`.

## Технологии

- **Язык**: Go (стандартная библиотека + минимальные зависимости)
- **База данных**: SQLite через `modernc.org/sqlite` (чистый Go, без CGO)
- **Фронтенд**: HTML5 Canvas, vanilla JavaScript
- **Аутентификация**: JWT (JSON Web Tokens) с хешированием паролей через bcrypt
- **Алгоритм сравнения**: Dynamic Time Warping (DTW)
- **Эталонные данные**: KanjiVG (SVG-файлы с траекториями черт)

## Зависимости Go

- `modernc.org/sqlite` — драйвер SQLite без CGO
- `golang.org/x/crypto` — хеширование паролей (bcrypt)
- `github.com/golang-jwt/jwt/v5` — работа с JWT-токенами
